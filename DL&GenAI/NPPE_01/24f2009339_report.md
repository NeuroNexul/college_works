## 1. Introduction
Chest X-ray imaging is a widely used diagnostic tool for detecting thoracic diseases such as pneumonia, edema, and cardiomegaly. However, **manual interpretation is time-consuming and subject to human variability**.

In this assignment, the problem is formulated as a **multi-class classification task**, where each chest X-ray image must be classified into one of **20 pathology classes**, including a "No Finding" category.

#### The Dataset
The dataset consists of:
- **51,043 training images of Chest X-Rays**.
- **17,015 test images of Chest X-Rays**.
- Labels provided in **one-hot encoded format**.

A key challenge in this problem is:
- **Class imbalance:** The provided data is highly imbalanced, with the majority of the samples (over 60%) from the "No Finding" category (19th index). However, the rare diseases appear very rarely.
- **Asymmetric misclassification costs**, where false negatives are heavily penalised (-5) compared to false positives (-1).

#### Objective
The objective is to build a deep learning model that:
- Accurately classifies chest X-ray images into the 20 given categories.
- Minimises false negatives `(The model predicts No Finding, but the patient actually has a disease)`.
- Maximises the competition score based on the given cost function.

### Pre-requisite
Before starting the experiment, I spent some time investigating potential challenges and exploring several solutions online.

#### I. Class Imbalance
Since we are provided with a medical dataset, having it be imbalanced is quite obvious. This means that there are some classes appearing very frequently (like "No Finding" or some common diseases). While others appear very rarely.

To solve this and generalise our model, we have a bunch of viable options:
- **Class weighting**
- **Oversampling rare classes**
- **Focal loss**
- **Balanced batch sampling**
But using multiple of these techniques altogether may cause our model to entirely ignore the common classes.

For this project, I'm using **WeightedRandomSampler** to do oversampling of the rare cases.

#### II. Data format
We were given the dataset as a bunch of Chest X-Rays inside `/images` folder and two CSV files as `train.csv` and `test.csv`.

The train dataset contains the file name of the X-Ray image and the respective class in one-hot encoded format.

```
id           Atelectasis    Cardiomegaly   ...   Calcification of the Aorta    No Finding
977df.png    0              0              ...   0                             1
```

Since each X-ray image is assigned to one of 20 classes, it is much cleaner to work with a label having the index of the assigned class.

So, I created a bunch of utility functions to convert one-hot to labels and vice versa.

```python
def onehot_to_label(onehot_vector):
    return int(np.argmax(onehot_vector))

def label_to_onehot(y, num_classes):
    return np.eye(num_classes)[[y]][0]

def labels_to_onehot(y, num_classes):
    return np.array([label_to_onehot(label, num_classes) for label in y])
```

#### III. Custom Metrics
The model is evaluated using a **macro-averaged asymmetric cost function**:

```
True Positive  -> +1  
False Positive -> -1  
False Negative -> -5  
```

So, to keep the experiment aligned with the final evaluation, I used the same metric to compare the results:

```python
def competition_score(y_true, y_pred, num_classes=num_classes):
    y_true = np.asarray(y_true)
    y_pred = np.asarray(y_pred)
    y_true = labels_to_onehot(y_true, num_classes) if y_true.ndim == 1 else y_true
    y_pred = labels_to_onehot(y_pred, num_classes) if y_pred.ndim == 1 else y_pred

    class_scores = []
    for c in range(num_classes):
        true_c = y_true[:, c]
        pred_c = y_pred[:, c]

        N_c = np.sum(true_c)
        if N_c == 0:
            class_scores.append(0.0)
            continue
        TP = np.sum((true_c == 1) & (pred_c == 1))
        FP = np.sum((true_c == 0) & (pred_c == 1))
        FN = np.sum((true_c == 1) & (pred_c == 0))
        class_scores.append((TP - FP - 5 * FN) / N_c)
    return np.mean(class_scores)
```

## 3. Methodology

#### Model Selection
I experimented with convolutional neural network (CNN) architectures, primarily:
- **ResNet 50**
- **ResNet 101**
- **DenseNet 121**
- **VGG16**

Pretrained models were chosen due to:
- Faster convergence
- Better feature extraction from medical images
- Reduced training time compared to training from scratch

I've utilised a transfer learning method for this. Below are the generic procedure.
- Initialised the model with pre-trained weights.
- Freeze the initial layers (Conv Layers) to hold the learnings from the Imagenet dataset.
  - Replaced the classifier / fully-connected layer with a custom one.
  - Then trained the classifier.
- Unfreeze all the layers.
  - Then train again to Fine-Tune the model.
 
```python
model = models.resnet101(weights=models.ResNet101_Weights)

# Freeze all layers
for name, param in model.named_parameters():
    if "layer4" not in name:
        param.requires_grad = False

# Replace the classifier
model.fc = nn.Sequential(
    nn.Linear(2048, 512),
    nn.ReLU(),
    nn.Dropout(0.5),
    nn.Linear(512, 20),
)

## Send the model to the device
model = model.to(device)
```

### Data Preprocessing
The following preprocessing steps were applied:
- **Image Resizing:** All images resized to 256×256, then **center cropped** to 224×224.
- **Normalisation:** Pixel values normalised using the standard mean and standard deviation according to the CNN architecture.
- **Tensor Conversion:** Images converted to tensors for model input.

### Data Augmentation
To improve generalisation and handle class imbalance:
- Random horizontal flipping  
- Rotation and affine transformations  
- Random brightness/contrast adjustments 

### Handling Class Imbalance
As mentioned above, to handle class imbalance, I've used **WeightedRandomSampler** in the **DataLoader**.

```python
class_counts = np.bincount(y_train)
class_weights = 1.0 / (class_counts + 1e-6)
class_weights = np.sqrt(class_weights)

sample_weights = class_weights[y_train]
sample_weights = torch.from_numpy(sample_weights).float()

sampler = WeightedRandomSampler(
    weights=sample_weights,
    num_samples=len(sample_weights),
    replacement=True
)
```

### Loss Function
Since I've already handled the class imbalance through oversampling, I'm using a balanced loss function. As confidence matters for different classes to be assigned, I'm using `BCEWithLogitsLoss`.

I've tried other loss functions as well, such as **Cross Entropy Loss**, **Weighted Cross Entropy Loss**, **Focal Loss**, and **Custom Asymmetric Loss**. But finally, `BCEWithLogitsLoss` gave me the best result.

### Why `BCEWithLogitsLoss` won?
`BCEWithLogitsLoss` prioritises the probabilities of different classes, which makes the cost more effective and updates the weights for the necessary changes only.

### Training Setup
- Optimiser: **Adam**
- Learning Rate: Tuned between **1e-3 to 1e-5**
- Batch Size: 32
- Train-validation split: **Stratified split (80:20)**

## 5. Conclusion
In this project, a deep learning-based system was developed to classify chest X-ray images into 20 pathology classes.

### Key Learnings:
- Class imbalance has a major impact on model performance.
- Standard accuracy is not a suitable metric for this problem.
- Minimising false negatives is critical due to the high penalty.
- Weighted loss functions are effective for medical datasets.
- Spent a lot of time fixing the model being stuck at local minima.

The extreme negative penalty causes the model to get optimised for a globally optimal solution, making it not learn anything. Had to use **BCEWithLogitsLoss** to glorify the individual class confidences.

### Future Improvements:
- Use advanced architectures (EfficientNet, DenseNet)
- Implement better focal or asymmetric loss
- Perform threshold tuning (For the majority class)
- Try model ensembling
